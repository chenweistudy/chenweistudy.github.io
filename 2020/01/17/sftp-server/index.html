    一.Sftp服务搭建步骤

    1.查看openssh的版本

     ssh –V  

如果低于4.8p1，需要自行升级安装，

    2.创建sftp组：

    groupadd sftp

    3.创建测试账户:

    (1)　如果用户在系统上不存在，使用以下命令创建

    创建一个sftp用户，用户名为mysftp，密码为mysftp

修改用户密码和修改Linux用户密码是一样的。

useradd -g sftp -s /bin/false mysftp  //用户名

passwd mysftp  //密码

    (2)对于已经存在的用户，使用以下usermod命令进行修改：

    usermod -g sftp -s /bin/false mysftp

    如果想要修改用户的默认家目录，那么可以在useradd和usermod命令中使用‘-d’选项，并设置合适的权限。

    4.创建目录：

    mkdir /opt/sftp   #cd /opt/sftp;     

mkdir testuser

    5.修改测试账户HOME路径：

    usermod -d /opt/sftp/testuser testuser

    6.配置sshd_config ：

    (1).文本编辑器打开 /etc/ssh/sshd_config

    vi /etc/ssh/sshd_config

    (2).找到如下这行，用#符号注释掉，大致在文件末尾处。

    #Subsystem   sftp    /usr/libexec/openssh/sftp-server

    7.添加如下几行

    Subsystem       sftp    internal-sftp  

Match Group sftp   //指定以下的行将仅仅匹配sftp组中的用户

ChrootDirectory /opt/sftp/%u  //重写根目录成登录用户的根目录下,参数指定用户验证后用于chroot环境的路径(默认的用户家目录)。对于用户 mysftp 该路径就是/data/sftp/mysftp

ForceCommand    internal-sftp   //使用sftp服务使用系统自带的internal-sftp

AllowTcpForwarding no  //是否允许TCP转发，默认值为"yes", 禁止TCP转发并不能增强安全性，除非禁止了用户###对shell的访问，因为用户可以安装他们自己的转发器

X11Forwarding no  //由于使用X11转发的可能带来的风险，此指令默认值为"no"

    8.设定Chroot目录权限：

    chown root:sftp /opt/sftp/testuser 
//chown命令由单词change owner组合而成。将目录 /data/sftp/mysftp的拥有者设为root，群体的使用者 sftp

chmod 755 /opt/sftp/testuser

    9.建立SFTP用户登入后可写入的目录

    按照上面设置后，在重启sshd服务后，用户mysftp已经可以登录。但使用chroot指定根目录后，根应该是无法写入的，所以要新建一个目录供mysftp上传文件。这个目录所有者为mysftp，所有组为sftp，所有者有写入权限，而所有组无写入权限。命令如下：

    mkdir /opt/sftp/testuser/upload  

chown uplus:sftp /opt/sftp/testuser/upload  

chmod 755 /opt/sftp/testuser/upload

    10.关闭防火墙

    修改/etc/selinux/config

    (1).永久性关闭防火墙。

    文本编辑器打开/etc/selinux/config

      vi /etc/selinux/config  

将文件中的SELINUX=enforcing 修改为 SELINUX=disabled ，然后保存。

//关闭selinux防火墙,永久性关闭。修改selinux的配置文件，重启后生效。

    (2).临时关闭防火墙。

    setenforce 0  //表示关闭selinux防火墙，临时关闭。重启后失效。

    11.重启sshd服务

    service sshd restart    重启sshd服务：

    12.测试

    用mysftp用户名登录，yes确定，回车输入密码。

sftp mysftp@127.0.0.1    // 显示 sftp> 则sftp搭建成功。
